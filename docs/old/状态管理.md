## 1. 技术选型

### 1.1 为什么选择 Provider？

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| Provider | 官方推荐、简单、性能好 | 大项目需要规范 | ✅ |
| Riverpod | 编译时安全、不依赖 Context | 学习曲线稍高 | ❌ |
| Bloc | 严格事件驱动、可测试性好 | 样板代码多 | ❌ |
| GetX | 简洁、功能全面 | 过度封装、争议多 | ❌ |
| MobX | 响应式、代码少 | 需要代码生成 | ❌ |

**选择理由**:
1. **官方推荐**: Flutter 团队官方推荐的状态管理方案
2. **学习成本低**: 基于 InheritedWidget，概念简单
3. **足够灵活**: 满足本项目需求
4. **生态成熟**: 大量文档和社区支持

### 1.2 Provider 类型

| 类型 | 用途 | 本项目使用 |
|------|------|------------|
| `ChangeNotifierProvider` | 可变状态，手动通知 | ✅ 主要使用 |
| `Provider` | 不变值或依赖注入 | ✅ 数据库注入 |
| `FutureProvider` | 异步数据 | ❌ |
| `StreamProvider` | 流数据 | ❌ |
| `ProxyProvider` | 依赖其他 Provider | ❌ |

## 3. 核心 Provider

### 3.1 ChatProvider

**职责**: 管理聊天会话和消息

**文件**: [`lib/core/providers/chat_provider.dart`](../lib/core/providers/chat_provider.dart)

```dart
class ChatProvider extends ChangeNotifier {
  final AppDatabase _db;
  
  // 状态
  List<ChatSession> _sessions = [];
  ChatSession? _currentSession;
  List<ChatMessage> _currentMessages = [];
  bool _isGenerating = false;
  
  // Getters
  List<ChatSession> get sessions => _sessions;
  ChatSession? get currentSession => _currentSession;
  List<ChatMessage> get currentMessages => _currentMessages;
  bool get isGenerating => _isGenerating;

  // 核心方法
  Future<void> loadSessions();
  Future<void> selectSession(String sessionId);
  Future<void> createSession(String roleId, String meId);
  Future<void> deleteSession(String sessionId);
  Future<void> addMessage(ChatMessage message);
  Future<void> generateAiResponse();
  Future<void> updateSessionSettings(...);
}
```

**状态更新流程**:

```
用户操作 (UI)
      │
      ▼
ChatProvider 方法调用
      │
      ├── 更新数据库 (持久化)
      │
      ├── 更新内存状态 (_sessions, _currentMessages 等)
      │
      └── notifyListeners() (通知 UI 更新)
```

## 4. 数据流模式

### 4.1 单向数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                           UI Layer                               │
│                                                                  │
│  Consumer<ChatProvider>(                                        │
│    builder: (context, provider, _) {                            │
│      return ListView(                                            │
│        children: provider.currentMessages.map(...),             │
│      );                                                          │
│    },                                                            │
│  )                                                               │
└────────────────────────────────┬────────────────────────────────┘
                                 │
                    ┌────────────┴────────────┐
                    │                         │
                    ▼ 读取                    │ 事件
              provider.messages              │
                    │                         │
                    │                         ▼
                    │              provider.addMessage(...)
                    │                         │
                    │                         ▼
┌───────────────────┴─────────────────────────────────────────────┐
│                        Provider Layer                            │
│                                                                  │
│  Future<void> addMessage(ChatMessage message) async {           │
│    // 1. 保存到数据库                                              │
│    await _db.insertMessage(message);                             │
│                                                                  │
│    // 2. 更新内存状态                                              │
│    _currentMessages.add(message);                                │
│                                                                  │
│    // 3. 通知 UI 更新                                              │
│    notifyListeners();                                            │
│  }                                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 异步操作处理

```dart
class ChatProvider extends ChangeNotifier {
  bool _isLoading = false;
  String? _error;

  bool get isLoading => _isLoading;
  String? get error => _error;

  Future<void> loadMessages() async {
    _isLoading = true;
    _error = null;
    notifyListeners();

    try {
      _currentMessages = await _db.getMessages(_currentSession!.id);
    } catch (e) {
      _error = e.toString();
    } finally {
      _isLoading = false;
      notifyListeners();
    }
  }
}

// UI 中使用
Consumer<ChatProvider>(
  builder: (context, provider, _) {
    if (provider.isLoading) {
      return CircularProgressIndicator();
    }
    if (provider.error != null) {
      return ErrorWidget(provider.error!);
    }
    return MessageList(provider.currentMessages);
  },
)
```

### 4.3 跨 Provider 通信

```dart
// 方式 1: 通过 Context 获取其他 Provider
class ChatProvider extends ChangeNotifier {
  final BuildContext _context;

  Future<void> _handleMemoryMessage(ChatMessage message) async {
    final memoryProvider = Provider.of<MemoryProvider>(_context, listen: false);
    await memoryProvider.addMemory(
      roleId: _currentSession!.roleId,
      content: message.content,
    );
  }
}

// 方式 2: 构造函数注入
class ChatProvider extends ChangeNotifier {
  final MemoryProvider _memoryProvider;

  ChatProvider(this._memoryProvider);
}

// 方式 3: 使用 ProxyProvider
MultiProvider(
  providers: [
    ChangeNotifierProvider(create: (_) => MemoryProvider()),
    ChangeNotifierProxyProvider<MemoryProvider, ChatProvider>(
      create: (_) => ChatProvider(null),
      update: (_, memoryProvider, chatProvider) {
        return chatProvider!..memoryProvider = memoryProvider;
      },
    ),
  ],
)
```

## 5. 依赖注入

### 5.1 数据库注入

```dart
// main.dart
MultiProvider(
  providers: [
    // 数据库作为根 Provider
    Provider<AppDatabase>(
      create: (_) => AppDatabase(),
      dispose: (_, db) => db.close(),
    ),
    
    // 其他 Provider 通过 context 获取数据库
    ChangeNotifierProvider(
      create: (context) => ChatProvider(

  ],
```

## 6. 最佳实践

### 6.1 UI 优化

```dart
// ❌ 不推荐: 整个页面监听
class ChatScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final provider = context.watch<ChatProvider>();  // 任何变化都重建整个页面
    return Column(
      children: [
        Header(),
        MessageList(provider.messages),
        InputBar(),
      ],
    );
  }
}

// ✅ 推荐: 精细化监听
class ChatScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Header(),  // 不需要监听
        Consumer<ChatProvider>(  // 只有 MessageList 监听
          builder: (_, provider, __) => MessageList(provider.messages),
        ),
        Selector<ChatProvider, bool>(  // 只监听特定字段
          selector: (_, p) => p.isGenerating,
          builder: (_, isGenerating, __) => InputBar(disabled: isGenerating),
        ),
      ],
    );
  }
}
```

### 6.3 避免不必要的通知

```dart
class ChatProvider extends ChangeNotifier {
  List<ChatMessage> _messages = [];

  // ❌ 不推荐: 每次都通知
  void updateMessage(ChatMessage message) {
    final index = _messages.indexWhere((m) => m.id == message.id);
    _messages[index] = message;
    notifyListeners();
  }

  // ✅ 推荐: 检查是否真的变化
  void updateMessage(ChatMessage message) {
    final index = _messages.indexWhere((m) => m.id == message.id);
    if (_messages[index] == message) return;  // 无变化
    _messages[index] = message;
    notifyListeners();
  }
}
```

### 6.4 错误处理

```dart
class ChatProvider extends ChangeNotifier {
  String? _error;
  String? get error => _error;

  void clearError() {
    _error = null;
    notifyListeners();
  }

  Future<void> doSomething() async {
    try {
      // 操作
    } catch (e) {
      _error = e.toString();
      notifyListeners();
      
      // 可选: 自动清除错误
      Future.delayed(Duration(seconds: 3), clearError);
    }
  }
}
```

## 7. 设计决策

### 7.1 为什么使用 ChangeNotifier？

**优点**:
- 简单直观
- 无需代码生成
- 灵活控制通知时机

**缺点**:
- 需要手动调用 `notifyListeners()`
- 大量状态时可能忘记通知

**替代方案考虑**:

| 方案 | 优点 | 缺点 |
|------|------|------|
| ValueNotifier | 自动通知 | 只能存储单个值 |
| StateNotifier | 不可变状态 | 需要额外依赖 |
| Riverpod | 更安全 | 学习成本高 |

### 7.2 为什么在 Provider 中直接调用数据库？

**方案对比**:

```dart
// 方案 1: Provider 直接调用数据库 ✅
class ChatProvider extends ChangeNotifier {
  final AppDatabase _db;
  
  Future<void> addMessage(ChatMessage message) async {
    await _db.insertMessage(message);
    _messages.add(message);
    notifyListeners();
  }
}

// 方案 2: 通过 Repository 层
class ChatRepository {
  Future<void> insertMessage(ChatMessage message);
}

class ChatProvider extends ChangeNotifier {
  final ChatRepository _repository;
  // ...
}
```

**选择方案 1 的原因**:
1. 项目规模适中，Repository 层增加复杂度
2. Drift 已经提供了很好的抽象
3. 减少代码层级

### 7.3 Provider 粒度

**原则**: 按功能领域划分，而非按页面划分

```dart
// ❌ 按页面划分
class ChatScreenProvider { ... }
class SettingsScreenProvider { ... }

// ✅ 按功能领域划分
class ChatProvider { ... }  // 聊天功能
class ContactProvider { ... }  // 联系人功能
class ApiSettingsProvider { ... }  // API 设置
```

### 7.4 初始化时机

```dart
// 方案 1: 应用启动时初始化所有
void main() async {
  final db = AppDatabase();
  final chatProvider = ChatProvider(db);
  await chatProvider.loadSessions();  // 启动时加载
  // ...
}

// 方案 2: 懒加载 ✅
class ChatProvider extends ChangeNotifier {
  bool _initialized = false;
  
  Future<void> ensureInitialized() async {
    if (_initialized) return;
    await loadSessions();
    _initialized = true;
  }
}
```

**选择方案 2 的原因**:
1. 加快应用启动速度
2. 按需加载，节省资源
3. 避免启动时的大量 I/O
