## 1. 概述

### 1.1 核心职责

消息解析器负责将 AI 返回的文本响应转换为结构化的 `ChatMessage` 列表。

```
AI 原始响应 (字符串)
        │
        ▼
┌───────────────────┐
│  ResponseParser   │
│                   │
│  - 预处理          │
│  - JSON 解析       │
│  - XML 解析        │
│  - 特殊消息处理     │
└───────────────────┘
        │
        ▼
List<ChatMessage>
```

### 1.2 核心文件

| 文件 | 职责 |
|------|------|
| [`lib/core/services/xml_parser.dart`](../lib/core/services/xml_parser.dart) | 主解析逻辑 |
| [`lib/core/models/regex_rule_model.dart`](../lib/core/models/regex_rule_model.dart) | 正则规则模型 |
| [`lib/core/providers/regex_settings_provider.dart`](../lib/core/providers/regex_settings_provider.dart) | 正则设置管理 |

### 2.2 核心方法签名

```dart
class ResponseParser {
  static Future<List<ChatMessage>> parse(
    String response, {
    required Map<String, String> idMapping,
    required bool enableExtendedChat,
    required bool enableEmoji,
    required bool enableTextToImage,
    required List<EmojiModel> availableEmojis,
    List<RegexRule>? regexRules,
  }) async {
    // 1. 预处理
    response = _preprocess(response, regexRules);

    // 2. 尝试 JSON 解析
    final jsonResult = _tryParseJson(response);
    if (jsonResult != null) {
      return _processItems(jsonResult, ...);
    }

    // 3. 尝试 XML 解析
    final xmlResult = _tryParseXml(response);
    if (xmlResult != null) {
      return _processItems(xmlResult, ...);
    }

    // 4. 无法解析，作为纯文本返回
    return [
      ChatMessage(
        type: MessageType.words,
        content: response.trim(),
      ),
    ];
  }
}
```

## 3. JSON 格式解析

### 3.1 期望格式

```json
[
  {"type": "word", "content": "你好呀！"},
  {"type": "action", "content": "*微笑着挥手*"},
  {"type": "thought", "content": "（希望他今天心情好）"},
  {"type": "emoji", "content": "开心"},
  {"type": "image", "content": "一个阳光明媚的公园场景"}
]
```

### 3.2 解析实现

```dart
List<Map<String, dynamic>>? _tryParseJson(String response) {
  // 方法 1: 直接解析
  try {
    final decoded = jsonDecode(response);
    if (decoded is List) {
      return _validateAndCast(decoded);
    }
  } catch (_) {}

  // 方法 2: 提取 JSON 代码块
  final jsonBlockRegex = RegExp(r'```json\s*([\s\S]*?)\s*```', caseSensitive: false);
  final blockMatch = jsonBlockRegex.firstMatch(response);
  if (blockMatch != null) {
    try {
      final decoded = jsonDecode(blockMatch.group(1)!);
      if (decoded is List) {
        return _validateAndCast(decoded);
      }
    } catch (_) {}
  }

  // 方法 3: 提取数组部分
  final arrayRegex = RegExp(r'\[[\s\S]*\]');
  final arrayMatch = arrayRegex.firstMatch(response);
  if (arrayMatch != null) {
    try {
      final decoded = jsonDecode(arrayMatch.group(0)!);
      if (decoded is List) {
        return _validateAndCast(decoded);
      }
    } catch (_) {}
  }

  return null;
}

List<Map<String, dynamic>>? _validateAndCast(List decoded) {
  final result = <Map<String, dynamic>>[];
  
  for (final item in decoded) {
    if (item is Map<String, dynamic>) {
      // 验证必要字段
      if (item.containsKey('type') && item.containsKey('content')) {
        result.add(item);
      }
    }
  }

  return result.isEmpty ? null : result;
}
```

### 3.3 支持的变体

解析器能处理多种 JSON 变体：

```json
// 变体 1: 标准格式
[{"type": "word", "content": "内容"}]

// 变体 2: 包含额外字段
[{"type": "word", "content": "内容", "emotion": "happy"}]

// 变体 3: type 使用不同命名
[{"type": "words", "content": "内容"}]  // 会标准化为 "word"
[{"type": "text", "content": "内容"}]   // 会标准化为 "word"

// 变体 4: 混合在其他文本中
这是我的回复：
```json
[{"type": "word", "content": "内容"}]
```
```

## 4. XML 格式解析

### 4.1 期望格式

```xml
<output>
  <words>你好呀！</words>
  <action>*微笑着挥手*</action>
  <thought>（希望他今天心情好）</thought>
  <emoji>开心</emoji>
  <image>一个阳光明媚的公园场景</image>
</output>
```

### 4.2 解析实现

```dart
List<Map<String, dynamic>>? _tryParseXml(String response) {
  // 方法 1: 直接解析
  try {
    final document = XmlDocument.parse(response);
    return _extractFromXml(document);
  } catch (_) {}

  // 方法 2: 包装 root 标签
  try {
    final wrapped = '<root>$response</root>';
    final document = XmlDocument.parse(wrapped);
    return _extractFromXml(document);
  } catch (_) {}

  // 方法 3: 修复常见错误后解析
  final cleaned = _cleanXml(response);
  try {
    final wrapped = '<root>$cleaned</root>';
    final document = XmlDocument.parse(wrapped);
    return _extractFromXml(document);
  } catch (_) {}

  return null;
}

List<Map<String, dynamic>>? _extractFromXml(XmlDocument document) {
  final result = <Map<String, dynamic>>[];
  
  // 遍历所有元素
  void processElement(XmlElement element) {
    for (final child in element.childElements) {
      final type = child.name.local;
      final content = child.innerText.trim();
      
      if (content.isNotEmpty) {
        result.add({
          'type': _normalizeType(type),
          'content': content,
        });
      }
      
      // 递归处理嵌套元素
      if (child.childElements.isNotEmpty) {
        processElement(child);
      }
    }
  }

  processElement(document.rootElement);
  return result.isEmpty ? null : result;
}

String _cleanXml(String xml) {
  // 移除 XML 声明
  xml = xml.replaceAll(RegExp(r'<\?xml[^>]*\?>'), '');
  
  // 转义未转义的 &
  xml = xml.replaceAllMapped(
    RegExp(r'&(?!(?:amp|lt|gt|quot|apos);)'),
    (m) => '&amp;',
  );
  
  // 移除无效字符
  xml = xml.replaceAll(RegExp(r'[\x00-\x08\x0B\x0C\x0E-\x1F]'), '');
  
  return xml;
}
```

### 4.3 类型标准化

```dart
String _normalizeType(String type) {
  switch (type.toLowerCase()) {
    // 文字消息
    case 'words':
    case 'word':
    case 'text':
    case 'message':
    case 'say':
      return 'word';
    
    // 动作
    case 'actions':
    case 'action':
    case 'act':
    case 'do':
      return 'action';
    
    // 想法
    case 'thoughts':
    case 'thought':
    case 'think':
    case 'mind':
      return 'thought';
    
    // 状态
    case 'states':
    case 'state':
    case 'status':
      return 'state';
    
    // 表情
    case 'emojis':
    case 'emoji':
    case 'sticker':
      return 'emoji';
    
    // 图片
    case 'images':
    case 'image':
    case 'picture':
    case 'pic':
      return 'image';
    
    // 记忆
    case 'memories':
    case 'memory':
    case 'remember':
      return 'memory';
    
    // 朋友圈
    case 'moments':
    case 'moment':
    case 'post':
      return 'moment';
    
    default:
      return type.toLowerCase();
  }
}
```

## 5. 特殊消息处理

### 5.1 图片生成

```dart
Future<ChatMessage?> _processImageMessage(
  String content,
  bool enableTextToImage,
) async {
  if (!enableTextToImage) {
    // 生图关闭时，作为普通文本返回
    return ChatMessage(
      type: MessageType.words,
      content: '(想发送一张图片：$content)',
    );
  }

  // 调用生图服务
  final result = await ImageGenerationService.generateImage(
    prompt: content,
  );

  if (result == null) {
    return ChatMessage(
      type: MessageType.words,
      content: '(图片生成失败：$content)',
    );
  }

  return ChatMessage(
    type: MessageType.image,
    content: result['path'],
    metadata: {
      'prompt': content,
      'api_preset_name': result['api_preset_name'],
      'style_preset_name': result['style_preset_name'],
      'paths': result['paths'],  // 可能有多张图
    },
  );
}
```

### 5.2 表情匹配

```dart
ChatMessage? _processEmojiMessage(
  String content,
  bool enableEmoji,
  List<EmojiModel> availableEmojis,
) {
  if (!enableEmoji || availableEmojis.isEmpty) {
    // 表情关闭时，作为普通文本返回
    return ChatMessage(
      type: MessageType.words,
      content: '[$content]',
    );
  }

  // 查找匹配的表情
  final emoji = _findEmoji(content, availableEmojis);
  
  if (emoji == null) {
    // 未找到匹配表情
    return ChatMessage(
      type: MessageType.words,
      content: '[$content]',
    );
  }

  return ChatMessage(
    type: MessageType.emoji,
    content: emoji.id,
    metadata: {
      'meaning': emoji.meaning,
      'matched_query': content,
    },
  );
}

EmojiModel? _findEmoji(String query, List<EmojiModel> emojis) {
  query = query.toLowerCase().trim();

  // 优先级 1: meaning 精确匹配
  for (final emoji in emojis) {
    if (emoji.meaning.toLowerCase() == query) {
      return emoji;
    }
  }

  // 优先级 2: meaning 包含查询
  for (final emoji in emojis) {
    if (emoji.meaning.toLowerCase().contains(query)) {
      return emoji;
    }
  }

  // 优先级 3: 查询包含 meaning
  for (final emoji in emojis) {
    if (query.contains(emoji.meaning.toLowerCase())) {
      return emoji;
    }
  }

  // 优先级 4: rawContent 匹配
  for (final emoji in emojis) {
    if (emoji.rawContent?.toLowerCase().contains(query) == true) {
      return emoji;
    }
  }

  return null;
}
```

## 6. 正则预处理

### 6.1 规则模型

```dart
class RegexRule {
  final String id;
  final String name;
  final String pattern;      // 正则表达式
  final String replacement;  // 替换内容
  final bool isEnabled;
  final int order;           // 执行顺序
}
```

### 6.2 预处理流程

```dart
String _preprocess(String response, List<RegexRule>? rules) {
  // 1. 应用用户自定义规则
  if (rules != null) {
    // 按 order 排序
    final sortedRules = [...rules]..sort((a, b) => a.order.compareTo(b.order));
    
    for (final rule in sortedRules) {
      if (!rule.isEnabled) continue;
      
      try {
        final regex = RegExp(rule.pattern);
        response = response.replaceAll(regex, rule.replacement);
      } catch (e) {
        // 无效正则，跳过
        AppLogService.w('Invalid regex rule ${rule.name}: $e');
      }
    }
  }

  // 2. 内置清理规则
  
  // 移除 Markdown 代码块标记（保留内容）
  response = response.replaceAllMapped(
    RegExp(r'```(?:json|xml)?\s*([\s\S]*?)\s*```'),
    (m) => m.group(1)!,
  );

  // 移除开头的说明文字
  response = response.replaceFirst(
    RegExp(r'^[^{\[<]*(?=[{\[<])'),
    '',
  );

  // 清理多余空行
  response = response.replaceAll(RegExp(r'\n{3,}'), '\n\n');

  return response.trim();
}
```

### 6.3 常用规则示例

```dart
// 规则 1: 移除思考过程
RegexRule(
  name: '移除思考标记',
  pattern: r'<think>[\s\S]*?</think>',

// 规则 2: 移除角色名前缀
RegexRule(

// 规则 3: 转换旧格式动作

  replacement: '',
)
```

## 7. 设计决策

### 7.1 为什么优先 JSON？

| 格式 | 优点 | 缺点 |
|------|------|------|
| JSON | 紧凑、解析快、现代 LLM 支持好 | 格式严格，容易出错 |
| XML | 容错好、层次清晰 | 冗余、Token 消耗高 |
| 纯文本 | 最简单 | 难以区分消息类型 |

**选择**: JSON 优先 → XML 回退 → 纯文本兜底

**理由**:
1. JSON 更节省 Token
2. 现代 LLM 输出 JSON 准确率高
3. XML 作为回退处理格式错误的情况
4. 纯文本确保不会完全丢失响应

### 7.2 为什么需要类型标准化？

**问题**: 不同 LLM 可能使用不同的类型名称

```json
// Claude 可能返回
{"type": "words", "content": "..."}

// GPT 可能返回
{"type": "text", "content": "..."}

// Gemini 可能返回
{"type": "message", "content": "..."}
```

**解决**: 统一的类型标准化函数

```dart
"words" → "word"
"text"  → "word"
"message" → "word"
```

### 7.3 为什么需要正则预处理？

**场景 1**: LLM 添加了不需要的说明

```
好的，我会用 JSON 格式回复：
[{"type": "word", "content": "你好"}]
```

**场景 2**: LLM 使用了旧格式

```
*微笑* 你好呀！
```

**场景 3**: LLM 添加了思考过程

```xml
<think>用户似乎心情不错...</think>
<words>你好呀！</words>
```

**解决**: 可配置的正则预处理规则

### 7.4 错误处理策略

```dart
// 核心原则：宁可降级，不可丢失

try {
  // 尝试完美解析
  return _parseStructured(response);
} catch (e) {
  // 降级为纯文本
  return [ChatMessage(type: MessageType.words, content: response)];
}
```

**保证**:
1. 任何输入都会有输出
2. 最差情况是作为纯文本显示
3. 不会因解析失败导致消息丢失

### 7.5 异步处理

```dart
// 图片生成是异步操作
static Future<List<ChatMessage>> parse(...) async {
  final messages = <ChatMessage>[];

  for (final item in parsedItems) {
    if (item['type'] == 'image') {
      // 异步等待图片生成
      final imageMessage = await _processImageMessage(item['content']);
      if (imageMessage != null) {
        messages.add(imageMessage);
      }
    } else {
      // 同步处理其他类型
      messages.add(_processOtherMessage(item));
    }
  }

  return messages;
}
```

**考虑**: 图片生成可能耗时较长，但保持顺序很重要，所以使用顺序 await 而非并行。
