## 1. 项目简介

LNPhone 是一款 **AI Native 手机系统应用**，模拟类似 iOS 风格的手机界面，用户可以与 AI 角色进行沉浸式聊天。核心功能包括：

- 🎭 **角色扮演聊天** - 与 AI 角色进行对话，支持多种消息类型
- 📱 **朋友圈** - 模拟社交媒体动态，角色可发布动态
- 🎨 **表情包系统** - 统一表情池，支持 AI 自动发送表情
- 🧠 **角色记忆** - 跨会话持久化记忆，增强沉浸感
- 🖼️ **文生图** - 集成多种生图 API，AI 可自动生成图片
- 🔔 **后台主动回复** - 后台服务定时检查，AI 主动发起对话
- 💰 **虚拟钱包** - 模拟红包、转账等资金往来

## 2. 技术栈

| 领域 | 技术选型 | 版本 | 选择原因 |
|------|----------|------|----------|
| 框架 | Flutter | 3.x | 跨平台、性能好、生态丰富 |
| 语言 | Dart | 3.x | Flutter 原生语言，强类型安全 |
| 状态管理 | Provider | 6.1.2 | 官方推荐、简单易用、性能好 |
| 数据库 | Drift (SQLite) | 2.24.2 | 类型安全、响应式、支持迁移 |
| HTTP | http | 1.6.0 | 官方包、轻量级、够用 |
| 后台服务 | flutter_background_service | 5.1.0 | 支持 Foreground Service |
| 通知 | flutter_local_notifications | 18.0.1 | 功能全面、维护活跃 |
| XML 解析 | xml | 6.5.0 | 用于解析 AI 结构化输出 |
| 图片处理 | image | 4.0.17 | 处理 GIF 透明通道等 |

### 为什么选择这些技术？

**Provider vs Riverpod vs Bloc**

| 方案 | 优点 | 缺点 | 我们的选择 |
|------|------|------|------------|
| Provider | 简单、官方推荐、学习曲线低 | 大型项目可能需要更多约束 | ✅ 选用 |
| Riverpod | 编译时安全、不依赖 Context | 学习曲线稍高 | ❌ |
| Bloc | 严格的事件驱动、可测试性好 | 样板代码多 | ❌ |

**选择理由**：Provider 足够满足本项目需求，且团队熟悉度高，开发效率优先。

**Drift vs sqflite vs Isar**

| 方案 | 优点 | 缺点 | 我们的选择 |
|------|------|------|------------|
| Drift | 类型安全、支持迁移、响应式 | 需要代码生成 | ✅ 选用 |
| sqflite | 轻量、无代码生成 | 手写 SQL、类型不安全 | ❌ |
| Isar | NoSQL、高性能 | 新项目、生态较小 | ❌ |

**选择理由**：本项目数据结构复杂（消息、联系人、朋友圈等），Drift 的类型安全和迁移支持极大降低了开发风险。

## 5. 核心模块

### 5.1 聊天系统

**文件**: [`lib/core/providers/chat_provider.dart`](../lib/core/providers/chat_provider.dart)

聊天系统是应用的核心，负责：
- 会话管理（创建、删除、置顶）
- 消息收发（支持 25 种消息类型）
- AI 回复生成（调用 LlmService）
- 消息持久化（存储到 Drift 数据库）

**消息类型** (详见 [`MessageType`](../lib/core/models/chat_model.dart:5)):

| 类型 | 用途 | 显示方式 |
|------|------|----------|
| words | 文字消息 | 气泡 |
| action | 动作描述 | 斜体/灰色 |
| thought | 内心想法 | 斜体 |
| state | 当前状态 | 标题栏更新 |
| emoji | 表情包 | 图片 |
| image | 图片 | 大图预览 |
| redpacket | 红包 | 红包卡片 |
| transfer | 转账 | 转账卡片 |
| moment | 朋友圈 | 不显示（添加到朋友圈） |
| ... | ... | ... |

### 5.2 LLM 集成

**文件**: [`lib/core/services/llm_service.dart`](../lib/core/services/llm_service.dart)

支持两种 LLM Provider：
- **OpenAI** (包括 OpenAI 兼容接口)
- **Gemini** (Google Generative AI)

**调用流程**:

```
1. ChatProvider.generateAiResponse()
      │
2.    ├── 构建历史消息（包含朋友圈虚拟消息）
      ├── 获取世界书、预设内容
      ├── 设置 ImageGenerationService 上下文
      │
3.    └── LlmService.generateResponse()
              │
4.            ├── _buildSystemPrompt()    // 构建系统提示词
              ├── _buildMessagesWithSimpleIds()  // 转换消息格式
              │
5.            └── _callOpenAI() / _callGemini()  // 调用 API
                      │
6.                    └── ResponseParser.parse()  // 解析响应
                              │
7.                            └── 处理生图、表情包等
```

### 5.3 响应解析

**文件**: [`lib/core/services/xml_parser.dart`](../lib/core/services/xml_parser.dart)

AI 返回的响应需要解析为结构化消息：

**支持格式**:
1. **JSON 数组** (优先)
   ```json
   [
     {"type": "word", "content": "你好！"},
     {"type": "action", "content": "*微笑*"}
   ]
   ```

2. **XML 格式** (兼容)
   ```xml
   <output>
     <words>你好！</words>
     <action>*微笑*</action>
   </output>
   ```

### 5.4 表情包系统

**文件**: [`lib/core/providers/emoji_provider.dart`](../lib/core/providers/emoji_provider.dart)

**统一池设计**:
- 所有表情存储在全局池中
- 角色通过 `subscribedGroupIds` 和 `subscribedEmojiIds` 订阅
- 支持"偷图"机制（自动添加到订阅列表）

**备份机制**:
- 使用 `backupPath` 存储备份文件路径（推荐）
- 弃用 `emojiData` 二进制存储（效率低）

### 5.5 后台服务

**文件**: [`lib/core/services/background_service.dart`](../lib/core/services/background_service.dart)

**功能**:
- 定时检查（每分钟）
- 主动回复触发
- 通知发送

**触发条件**:
1. 后台主动回复开关开启
2. 用户不活跃时间 > 设定阈值
3. 会话最后消息时间 > 设定阈值

## 6. 数据流

### 6.1 用户发送消息

```
用户输入
    │
    ▼
ChatScreen.onSend()
    │
    ▼
ChatProvider.addMessage()
    │
    ├── 保存消息到数据库
    │
    └── ChatProvider.generateAiResponse()
            │
            ├── 构建上下文
            ├── 调用 LlmService
            ├── 解析响应
            ├── 处理特殊消息（记忆、朋友圈等）
            │
            └── 添加 AI 消息到数据库
                    │
                    └── notifyListeners()
                            │
                            └── UI 更新
```

### 6.2 后台主动回复

```
Timer (每分钟)
    │
    ▼
BackgroundService._checkAndTriggerActiveReply()
    │
    ├── 检查开关和时间阈值
    ├── 遍历所有会话
    │
    └── BackgroundService._triggerAiReply()
            │
            ├── 加载 PromptConfig
            ├── 获取 ApiPreset
            ├── 调用 LlmService
            │
            └── 保存消息并发送通知
```

## 7. 设计决策与权衡

### 7.1 数据库设计：Blob vs 文件

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| Blob 存储 | 数据库事务一致性 | 数据库膨胀、内存压力 | 仅用于小数据备份 |
| 文件存储 | 性能好、易管理 | 需要额外维护路径 | ✅ 主要方案 |

**实现**: 
- 使用 `localPath` 存储文件相对路径
- 使用 `backupPath` 存储备份文件路径
- 弃用 `emojiData`/`avatarData` 等 Blob 字段

### 7.2 外键约束：启用 vs 禁用

```dart
// database.dart
// 不启用外键约束，避免 insertOrReplace 触发级联删除导致消息丢失
// db.execute('PRAGMA foreign_keys = ON;');
```

**原因**:
- Drift 的 `insertOrReplace` 会先删除再插入
- 启用外键约束后，会级联删除关联的 `ChatMessages`
- 这导致了严重的 Bug（消息莫名消失）

**解决方案**:
1. 禁用外键约束
2. 使用 `update()` 而非 `insertOrReplace()` 更新会话

### 7.3 消息 ID 简化策略

**问题**: AI 难以正确引用长 ID（如 `1738552800000-12345`）

**解决方案**: 使用简化 ID 映射

```dart
// 简化ID: 0001, 0002, 0003...
final idMapping = <String, String>{}; // simpleId -> realId

// 发送给 AI 时使用简化 ID
// 解析 AI 响应时通过 idMapping 还原
```

### 7.4 GIF 透明通道处理

**问题**: `File.copy()` 在某些 Android 版本上损坏 GIF 透明通道

**解决方案**: 使用字节复制

```dart
// ❌ 不推荐
await sourceFile.copy(targetPath);

// ✅ 推荐
final bytes = await sourceFile.readAsBytes();
await File(targetPath).writeAsBytes(bytes);
```
