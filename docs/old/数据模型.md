## 1. 聊天模型

**文件**: [`lib/core/models/chat_model.dart`](../lib/core/models/chat_model.dart)

### 1.1 ChatSession（聊天会话）

会话是用户与角色之间的一次对话上下文。

```dart
class ChatSession {
  final String id;                        // 唯一标识符
  final String roleId;                    // 关联的角色 ID
  final String meId;                      // 关联的用户人设 ID
  final int lastUpdated;                  // 最后更新时间戳（毫秒）
  final bool enableExtendedChat;          // 是否启用扩展聊天（动作/心理）
  final bool enableTextToImage;           // 是否启用文生图
  final bool enableEmoji;                 // 是否启用表情包
  final bool enableIndependentSendButton; // 是否启用独立发送/续写按钮
  final String? currentState;             // 当前状态描述
  final bool isPinned;                    // 是否置顶
  final String? apiPresetId;              // 文本 API 预设 ID
  final String? imageApiPresetId;         // 图片 API 预设 ID
  final List<String> worldInfoIds;        // 世界书 ID 列表
  final List<String> textPresetIds;       // 文本预设 ID 列表
  final String? backgroundImage;          // 背景图路径
}
```

**设计说明**:

| 字段 | 设计原因 |
|------|----------|
| `roleId` + `meId` | 支持用户切换不同"身份"与角色对话，增强角色扮演体验 |
| `enableExtendedChat` | 控制 AI 是否输出动作描写和心理活动 |
| `worldInfoIds` | 支持多个世界书组合，提供丰富的世界观设定 |
| `backgroundImage` | 个性化聊天界面，增强沉浸感 |

### 1.2 ChatMessage（聊天消息）

```dart
class ChatMessage {
  final String id;             // 唯一标识符

  final bool isMe;             // 是否为用户发送

  final String content;        // 消息内容
  final int timestamp;         // 时间戳（毫秒）
  final Map<String, dynamic>? metadata;  // 扩展元数据

}
```

### 1.3 MessageType（消息类型枚举）

```dart
enum MessageType {
  words,           // 0  - 文字消息
  action,          // 1  - 动作描述
  thought,         // 2  - 内心想法
  state,           // 3  - 当前状态
  emoji,           // 4  - 表情包
  image,           // 5  - 图片
  location,        // 6  - 位置分享
  redpacket,       // 7  - 红包
  transfer,        // 8  - 转账
  acceptRedpacket, // 9  - 接受红包
  rejectRedpacket, // 10 - 拒绝红包
  acceptTransfer,  // 11 - 接受转账
  rejectTransfer,  // 12 - 拒绝转账
  product,         // 13 - 商品推荐
  link,            // 14 - 链接分享
  note,            // 15 - 备忘提醒
  anniversary,     // 16 - 纪念日卡片
  memory,          // 17 - 记忆
  diary,           // 18 - 日记
  moment,          // 19 - 朋友圈
  momentComment,   // 20 - 朋友圈评论
  momentLike,      // 21 - 朋友圈点赞
  scene,           // 22 - 场景描述
  narration,       // 23 - 旁白
  options,         // 24 - 互动选项
}
```

**设计理念**:

为什么设计这么多消息类型？

| 类别 | 消息类型 | 目的 |
|------|----------|------|
| **基础对话** | words, action, thought | 提供丰富的角色扮演表达方式 |
| **媒体** | image, emoji, location | 模拟真实聊天应用功能 |
| **资金** | redpacket, transfer, accept/reject | 模拟微信红包/转账功能 |
| **社交** | moment, momentComment, momentLike | 联动朋友圈功能 |
| **剧情** | scene, narration, options | 支持视觉小说式的剧情推进 |

**替代方案对比**:

| 方案 | 优点 | 缺点 | 我们的选择 |
|------|------|------|------------|
| 使用枚举 | 类型安全、IDE 提示 | 扩展需改代码 | ✅ |
| 使用字符串 | 灵活、易扩展 | 无类型检查、易出错 | ❌ |
| 使用类继承 | 面向对象、可扩展 | 复杂度高、序列化麻烦 | ❌ |

**字段说明**:

| `subscribedGroupIds` | 角色可以使用的表情包分组 |
| `subscribedEmojiIds` | 角色单独订阅的表情（不在分组中） |

### 2.2 ContactMe（用户人设）

```dart
class ContactMe {
  final String id;                      // 唯一标识符
```

- **生图一致性**: 生成用户出现的图片时使用外貌描述

### 3.4 统一池设计原理

**问题**: 早期设计中，每个角色有独立的表情存储，导致：
- 重复存储相同表情
- 难以实现"偷图"功能
- 管理复杂

**解决方案**: 统一表情池 + 订阅模型

```
┌─────────────────────────────────────────────────────┐
│                    统一表情池                        │
│  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐    │
│  │Emoji 1 │  │Emoji 2 │  │Emoji 3 │  │Emoji 4 │    │
│  │(全局)  │  │(全局)  │  │(角色A) │  │(角色B) │    │
│  └────────┘  └────────┘  └────────┘  └────────┘    │
└─────────────────────────────────────────────────────┘
        │              │              │
        ▼              ▼              ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐
│     角色A     │ │     角色B     │ │     角色C     │
│ subscribed:   │ │ subscribed:   │ │ subscribed:   │
│ [1, 2, 3]     │ │ [1, 2, 4]     │ │ [1, 2]        │
└───────────────┘ └───────────────┘ └───────────────┘
```

**ID 生成策略**:

```dart
String generateNewEmojiId() {
  // 1. 查找已删除的 ID（回收利用）
  // 2. 如果没有，使用最大 ID + 1
  // 格式: emoji-id-00001, emoji-id-00002, ...
}
```

### 4.5 朋友圈与聊天的联动

**设计目标**: AI 发布朋友圈后，用户可以在朋友圈中看到；朋友圈的互动也可以在聊天中反映。

**实现方式**:

1. AI 返回 `moment` 类型消息时，自动创建朋友圈动态
2. 用户点赞/评论时，创建对应的聊天消息
3. 构建 LLM 上下文时，将朋友圈互动作为"虚拟消息"加入

```dart
// 虚拟消息示例
[
  {
    "type": "moment",
    "content": "角色发布了朋友圈：今天天气真好！"
  },
  {
    "type": "momentLike",
    "content": "用户给这条朋友圈点赞了"
  },
  {
    "type": "momentComment",
    "content": "用户评论了：确实！"
  }
]
```

### 5.3 记忆系统设计

**为什么需要记忆系统？**

- **跨会话持久化**: 聊天历史可能被清理，但重要信息需要保留
- **增强沉浸感**: AI 能"记住"用户的喜好、重要日期等
- **减少 Token 消耗**: 只发送精炼的记忆，而非完整历史

**记忆的添加方式**:

1. **AI 主动添加**: 通过 `memory` 类型消息
2. **用户手动添加**: 在记忆管理界面
3. **系统自动提取**: 未来可能实现

**记忆的使用方式**:

```dart
// 构建系统提示词时加入记忆
String _buildSystemPrompt(...) {
  // ...
  if (memories.isNotEmpty) {
    prompt += '\n\n### 关于用户的记忆\n';
    for (final memory in memories) {
      prompt += '- ${memory.content}\n';
    }
  }
  // ...
}
```

```dart
  redpacket,  // 红包
}

enum WalletTransactionDirection {
  income,   // 收入
  expense,  // 支出
}
```

### 6.3 钱包功能设计

**功能说明**:
- 模拟微信红包和转账功能
- 虚拟货币，无真实价值
- 增强角色扮演的沉浸感

**交互流程**:

```
用户发送红包
    │
    ▼
创建 redpacket 消息 ──► 扣款（expense）
    │
    ▼
AI 接收/拒绝
    │
    ├── 接收 ──► 创建 acceptRedpacket 消息 ──► 无操作（已扣）
    │
    └── 拒绝 ──► 创建 rejectRedpacket 消息 ──► 退款（income）
```

### 7.4 设计说明

**为什么需要 API 预设？**

- **多供应商支持**: 用户可以选择不同的 AI 供应商
- **配置隔离**: 每个预设独立配置，便于切换
- **敏感信息保护**: API Key 不导出

**供应商差异处理**:

| 供应商 | 认证方式 | 特殊处理 |
|--------|----------|----------|
| OpenAI | Bearer Token | 标准 chat/completions |
| Gemini | URL 参数 | 需要特殊请求格式 |
| Volcengine | Bearer Token | 图片 API 需要特殊字段 |
| Groklike | Bearer Token | 图片通过 chat/completions |
