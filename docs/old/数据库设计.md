## 1. 技术选型

### 1.1 为什么选择 Drift？

**Drift** (原名 Moor) 是 Flutter 生态中最成熟的 SQLite 封装库。

| 对比项 | Drift | sqflite | Isar | Hive |
|--------|-------|---------|------|------|
| 类型安全 | ✅ 编译时检查 | ❌ 手写 SQL | ✅ 代码生成 | ✅ |
| 关系型数据 | ✅ 原生支持 | ✅ 手动实现 | ⚠️ 有限支持 | ❌ |
| 迁移支持 | ✅ 强大 | ⚠️ 需手动 | ⚠️ 有限 | ❌ |
| 响应式查询 | ✅ 内置 | ❌ 需封装 | ✅ | ✅ |
| 学习曲线 | 中等 | 低 | 低 | 低 |
| 代码生成 | 需要 | 不需要 | 需要 | 可选 |

**选择 Drift 的理由**:

1. **数据结构复杂**: 本项目有 14+ 表，表间关系复杂，关系型数据库更合适
2. **类型安全**: 编译时检查 SQL，减少运行时错误
3. **迁移友好**: 内置版本迁移，支持复杂的 schema 变更
4. **响应式**: 支持 `watchSingle()`、`watch()` 等响应式查询

    },
  );
}

## 4. 类型转换器

Drift 支持自定义 TypeConverter，将复杂类型映射到数据库列。

### 4.1 StringListConverter（字符串列表）

```dart
class StringListConverter extends TypeConverter<List<String>, String> {
  const StringListConverter();

  @override
  List<String> fromSql(String fromDb) {
    if (fromDb.isEmpty || fromDb == '[]') return [];
    try {
      return (jsonDecode(fromDb) as List).cast<String>();
    } catch (e) {
      return [];
    }
  }

  @override
  String toSql(List<String> value) {
    return jsonEncode(value);
  }
}
```

**使用场景**: `worldInfoIds`、`textPresetIds`、`referenceImages` 等。

### 4.2 JsonMapConverter（JSON 对象）

```dart
class JsonMapConverter extends TypeConverter<Map<String, dynamic>, String> {
  const JsonMapConverter();

  @override
  Map<String, dynamic> fromSql(String fromDb) {
    if (fromDb.isEmpty) return {};
    try {
      return jsonDecode(fromDb) as Map<String, dynamic>;
    } catch (e) {
      return {};
    }
  }

  @override
  String toSql(Map<String, dynamic> value) {
    return jsonEncode(value);
  }
}
```

**使用场景**: `ChatMessage.metadata`、`ApiPreset.params` 等。

### 4.3 MomentsUserConverter（朋友圈用户）

```dart
class MomentsUserConverter extends TypeConverter<MomentsUser, String> {
  const MomentsUserConverter();

  @override
  MomentsUser fromSql(String fromDb) {
    final map = jsonDecode(fromDb) as Map<String, dynamic>;
    return MomentsUser.fromJson(map);
  }

  @override
  String toSql(MomentsUser value) {
    return jsonEncode(value.toJson());
  }
}
```

**使用场景**: `MomentsPost.user`。

## 5. 数据库迁移

### 5.1 迁移策略

```dart
@override
MigrationStrategy get migration => MigrationStrategy(
  onCreate: (Migrator m) => m.createAll(),
  onUpgrade: (Migrator m, int from, int to) async {
    // 按版本逐步迁移
    if (from < 30) {
      await m.addColumn(chatSessions, chatSessions.enableEmoji);
    }
    if (from < 31) {
      await m.addColumn(contactRoles, contactRoles.subscribedGroupIds);
      await m.addColumn(contactRoles, contactRoles.subscribedEmojiIds);
    }
    if (from < 32) {
      await m.addColumn(emojis, emojis.backupPath);
    }
    // ... 更多迁移
  },
  beforeOpen: (details) async {
    // 数据库打开前的操作
    // 注意: 我们不启用外键约束
    // await customStatement('PRAGMA foreign_keys = ON');
  },
);
```

### 5.2 迁移历史

| 版本 | 变更 | 原因 |
|------|------|------|
| 30 | 添加 `enableEmoji` | 支持表情包开关 |
| 31 | 添加 `subscribedGroupIds`/`subscribedEmojiIds` | 统一表情池订阅模型 |
| 32 | 添加 `emojis.backupPath` | 文件备份路径 |
| 33 | 添加 `emojis.type`/`roleId` | 区分全局和角色表情 |
| 34 | 添加 `emojiGroups.type`/`roleId` | 区分全局和角色分组 |
| 35 | 添加 `enableIndependentSendButton` | 独立发送/续写按钮 |
| 36 | 添加 `imageApiPresetId` | 独立的图片 API 预设 |

### 5.3 迁移最佳实践

1. **永远不删除列**: SQLite 不支持 DROP COLUMN（除非重建表）
2. **新增列设置默认值**: 避免 NOT NULL 约束失败
3. **逐版本迁移**: 确保任意版本都能升级到最新
4. **测试降级**: 虽然不推荐，但要处理降级场景

```dart
// 安全的列添加示例
if (from < 32) {
  try {
    await m.addColumn(emojis, emojis.backupPath);
  } catch (e) {
    // 列可能已存在（之前迁移失败后重试）
    print('Column might already exist: $e');
  }
}
```

## 6. 关键设计决策

### 6.1 外键约束：禁用

```dart
// ❌ 我们不启用这个
// await customStatement('PRAGMA foreign_keys = ON');
```

**问题背景**:

Drift 的 `insertOrReplace()` 实现是：
1. DELETE 现有记录
2. INSERT 新记录

启用外键约束后，DELETE 会触发 `ON DELETE CASCADE`，导致关联的 `ChatMessages` 被删除。

**解决方案**:

1. 禁用外键约束
2. 更新操作使用 `update()` 而非 `insertOrReplace()`

```dart
// ✅ 正确的更新方式
await (db.update(db.chatSessions)
  ..where((tbl) => tbl.id.equals(session.id)))
  .write(ChatSessionsCompanion(
    lastUpdated: Value(DateTime.now().millisecondsSinceEpoch),
  ));
```

### 6.2 Blob vs 文件路径

**演进过程**:

| 阶段 | 存储方式 | 问题 |
|------|----------|------|
| v1 | `emojiData` Blob | 数据库膨胀、内存压力 |
| v2 | `localPath` 文件路径 | 文件可能丢失 |
| v3 | `localPath` + `backupPath` | ✅ 最佳方案 |

**当前策略**:

```dart
// 表情存储
class Emojis extends Table {
  TextColumn get localPath => text().nullable()();    // 主路径
  TextColumn get backupPath => text().nullable()();   // 备份路径
  BlobColumn get emojiData => blob().nullable()();    // 已弃用，保留兼容
}
```

### 6.3 主键策略

**方案对比**:

| 方案 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| 自增 ID | 1, 2, 3 | 简单、有序 | 合并冲突、隐私泄露 |
| UUID | `550e8400-e29b...` | 全局唯一 | 长、不易读 |
| 时间戳 ID | `1738552800000` | 有序、简单 | 并发冲突 |
| 复合 ID | `role_abc123` | 可读、有意义 | 需要规范 |

**我们的选择**: 复合 ID 模式

```dart
// 会话 ID
final sessionId = 'session_${DateTime.now().millisecondsSinceEpoch}';

// 消息 ID
final messageId = '${DateTime.now().millisecondsSinceEpoch}-${Random().nextInt(10000)}';

// 表情 ID（可回收）
final emojiId = 'emoji-id-${paddedNumber}';  // emoji-id-00001
```

## 7. 性能优化

### 7.1 索引策略

虽然 Drift 表定义中没有显式索引，但应考虑添加：

```dart
// 推荐的索引（通过迁移添加）
await customStatement('''
  CREATE INDEX IF NOT EXISTS idx_messages_session 
  ON chat_messages(session_id)
''');

await customStatement('''
  CREATE INDEX IF NOT EXISTS idx_messages_timestamp 
  ON chat_messages(timestamp DESC)
''');
```

### 7.2 查询优化

**分页查询**:

```dart
// ✅ 使用 LIMIT + OFFSET
Future<List<ChatMessage>> getMessages(String sessionId, {
  int limit = 50,
  int offset = 0,
}) {
  return (select(chatMessages)
    ..where((m) => m.sessionId.equals(sessionId))
    ..orderBy([(m) => OrderingTerm.desc(m.timestamp)])
    ..limit(limit, offset: offset))
    .get();
}
```

**延迟加载**:

```dart
// ❌ 一次加载所有消息
final allMessages = await getAllMessages(sessionId);

// ✅ 只加载最近的消息，按需加载更多
final recentMessages = await getMessages(sessionId, limit: 50);
```

### 7.3 响应式查询

```dart
// 监听会话列表变化
Stream<List<ChatSession>> watchAllSessions() {
  return (select(chatSessions)
    ..orderBy([(s) => OrderingTerm.desc(s.lastUpdated)]))
    .watch();
}

// 在 Provider 中使用
class ChatProvider extends ChangeNotifier {
  StreamSubscription? _subscription;

  void _startWatching() {
    _subscription = db.watchAllSessions().listen((sessions) {
      _sessions = sessions;
      notifyListeners();
    });
  }
}
```

### 7.4 事务处理

```dart
// 批量操作使用事务
Future<void> deleteSessionWithMessages(String sessionId) async {
  await db.transaction(() async {
    await (delete(chatMessages)
      ..where((m) => m.sessionId.equals(sessionId)))
      .go();
    await (delete(chatSessions)
      ..where((s) => s.id.equals(sessionId)))
      .go();
  });
}
```
