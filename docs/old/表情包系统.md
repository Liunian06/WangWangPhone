## 1. 系统概述

### 1.1 功能特点

- 🎨 **统一表情池** - 所有表情存储在全局池中，角色通过订阅使用
- 📁 **分组管理** - 表情可按分组组织，支持全局分组和角色分组
- 🤖 **AI 集成** - AI 可以在对话中发送表情
- 🔄 **偷图机制** - 自动将 AI 使用的表情添加到角色订阅
- 💾 **文件备份** - 完善的文件备份和恢复机制
- 🏷️ **语义标注** - 每个表情有含义标注，便于 AI 匹配

### 1.2 核心文件

| 文件 | 职责 |
|------|------|
| [`lib/core/models/emoji_model.dart`](../lib/core/models/emoji_model.dart) | 表情数据模型 |

## 3. 统一池架构

### 3.1 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                      EmojiProvider                           │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                 _allEmojis (统一池)                  │    │
│  │     Map<String, EmojiModel> 所有表情                 │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                 _allGroups (分组池)                  │    │
│  │     Map<String, EmojiGroupModel> 所有分组            │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                              │
│  核心方法:                                                   │
│  - getAvailableEmojis(roleId) → 获取角色可用表情            │
│  - addEmojiToPool(...) → 添加表情到池                       │
│  - subscribeEmoji(roleId, emojiId) → 订阅表情              │
│  - unsubscribeEmoji(roleId, emojiId) → 取消订阅            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 获取可用表情

```dart
/// 获取角色可用的表情列表
List<EmojiModel> getAvailableEmojis(String roleId) {
  final role = _contactProvider.getRoleById(roleId);
  if (role == null) return [];

  final availableEmojis = <EmojiModel>[];

  // 1. 添加订阅分组中的表情
  for (final groupId in role.subscribedGroupIds) {
    final group = _allGroups[groupId];
    if (group != null) {
      // 获取该分组下的所有表情
      final groupEmojis = _allEmojis.values
          .where((e) => e.groupId == groupId)
          .toList();
      availableEmojis.addAll(groupEmojis);
    }
  }

  // 2. 添加单独订阅的表情
  for (final emojiId in role.subscribedEmojiIds) {
    final emoji = _allEmojis[emojiId];
    if (emoji != null && !availableEmojis.contains(emoji)) {
      availableEmojis.add(emoji);
    }
  }

  // 3. 添加角色专属表情
  final roleEmojis = _allEmojis.values
      .where((e) => e.type == EmojiType.role && e.roleId == roleId)
      .toList();
  for (final emoji in roleEmojis) {
    if (!availableEmojis.contains(emoji)) {
      availableEmojis.add(emoji);
    }
  }

  return availableEmojis;
}
```

### 3.3 ID 生成与回收

```dart
/// 生成新的表情 ID
String generateNewEmojiId() {
  // 1. 查找可回收的 ID
  final usedIds = _allEmojis.keys
      .map((id) => int.tryParse(id.replaceFirst('emoji-id-', '')))
      .whereType<int>()
      .toSet();

  // 2. 从 1 开始查找空缺
  for (var i = 1; i <= usedIds.length + 1; i++) {
    final candidateId = 'emoji-id-${i.toString().padLeft(5, '0')}';
    if (!usedIds.contains(i)) {
      return candidateId;
    }
  }

  // 3. 返回最大值 + 1
  final maxId = usedIds.isEmpty ? 0 : usedIds.reduce(max);
  return 'emoji-id-${(maxId + 1).toString().padLeft(5, '0')}';
}
```

**回收示例**:
```
已有 ID: 00001, 00002, 00004, 00005
删除后: 00001, 00002, 00005
新增时: 使用 00003 (回收)
再新增: 使用 00004 (回收)
再新增: 使用 00006 (新分配)
```


### 4.2 备份服务

**文件**: [`lib/core/services/emoji_file_backup_service.dart`](../lib/core/services/emoji_file_backup_service.dart)

```dart
class EmojiFileBackupService {
  static const String backupDirName = 'emoji_backup';

  /// 创建备份
  static Future<String?> createBackup(String emojiId, String sourcePath) async {
    final appDir = await getApplicationDocumentsDirectory();
    final backupDir = Directory('${appDir.path}/$backupDirName');
    
    if (!await backupDir.exists()) {
      await backupDir.create(recursive: true);
    }

    final extension = path.extension(sourcePath);
    final backupPath = '${backupDir.path}/$emojiId$extension';

    // 使用字节复制保留 GIF 透明通道
    final sourceFile = File(sourcePath);
    if (await sourceFile.exists()) {
      final bytes = await sourceFile.readAsBytes();
      await File(backupPath).writeAsBytes(bytes);
      return backupPath;
    }

    return null;
  }

  /// 从备份恢复
  static Future<bool> restoreFromBackup(String backupPath, String targetPath) async {
    final backupFile = File(backupPath);
    if (!await backupFile.exists()) {
      return false;
    }

    final bytes = await backupFile.readAsBytes();
    await File(targetPath).writeAsBytes(bytes);
    return true;
  }

  /// 删除备份
  static Future<void> deleteBackup(String backupPath) async {
    final file = File(backupPath);
    if (await file.exists()) {
      await file.delete();
    }
  }
}
```

### 4.3 GIF 透明通道保护

**问题**: `File.copy()` 在某些 Android 版本上会损坏 GIF 透明通道

**解决方案**: 使用字节复制

```dart
// ❌ 不推荐 - 可能损坏 GIF
await sourceFile.copy(targetPath);

// ✅ 推荐 - 字节级复制
final bytes = await sourceFile.readAsBytes();
await File(targetPath).writeAsBytes(bytes);
```

### 4.4 存储演进历史

| 版本 | 存储方式 | 问题 |
|------|----------|------|
| v1 | `emojiData` Blob | 数据库膨胀、内存压力大 |
| v2 | `localPath` 文件 | 文件可能意外丢失 |
| v3 | `localPath` + `backupPath` | ✅ 当前方案 |

### 5.2 表情匹配逻辑

```dart
// ResponseParser._findEmoji()
EmojiModel? _findEmoji(String meaning, List<EmojiModel> emojis) {
  // 1. 精确匹配 meaning
  for (final emoji in emojis) {
    if (emoji.meaning.toLowerCase() == meaning.toLowerCase()) {
      return emoji;
    }
  }

  // 2. 包含匹配
  for (final emoji in emojis) {
    if (emoji.meaning.toLowerCase().contains(meaning.toLowerCase())) {
      return emoji;
    }
  }

  // 3. rawContent 匹配
  for (final emoji in emojis) {
    if (emoji.rawContent?.toLowerCase().contains(meaning.toLowerCase()) == true) {
      return emoji;
    }
  }

  return null;
}
```

### 5.3 偷图机制

当 AI 使用了一个不在角色订阅列表中的表情时，自动添加到订阅：

```dart
// ChatProvider._processEmojiMessage()
Future<void> _processEmojiMessage(ChatMessage message, String roleId) async {
  final emojiId = message.content;
  final emoji = _emojiProvider.getEmojiById(emojiId);
  
  if (emoji != null) {
    final role = _contactProvider.getRoleById(roleId);
    if (role != null && !role.subscribedEmojiIds.contains(emojiId)) {
      // 自动订阅（偷图）
      await _contactProvider.subscribeEmoji(roleId, emojiId);
      
      // 可选: 显示提示
      _showSnackbar('已自动添加表情到 ${role.name} 的收藏');
    }
  }
}
```

## 6. 关键代码

### 6.1 添加表情

```dart
Future<String?> addEmoji({
  required String meaning,
  String? rawContent,
  required String sourcePath,
  String? groupId,
  EmojiType type = EmojiType.global,
  String? roleId,
}) async {
  // 1. 生成 ID
  final id = generateNewEmojiId();

  // 2. 复制文件到应用目录
  final appDir = await getApplicationDocumentsDirectory();
  final extension = path.extension(sourcePath);
  final localPath = '${appDir.path}/emojis/$id$extension';
  
  final bytes = await File(sourcePath).readAsBytes();
  await File(localPath).create(recursive: true);
  await File(localPath).writeAsBytes(bytes);

  // 3. 创建备份
  final backupPath = await EmojiFileBackupService.createBackup(id, localPath);

  // 4. 保存到数据库
  final emoji = EmojiModel(
    id: id,
    meaning: meaning,
    rawContent: rawContent,
    groupId: groupId,
    localPath: localPath,
    backupPath: backupPath,
    type: type,
    roleId: roleId,
    createdAt: DateTime.now().millisecondsSinceEpoch,
  );

  await db.into(db.emojis).insert(emoji.toCompanion());

  // 5. 更新内存缓存
  _allEmojis[id] = emoji;
  notifyListeners();

  return id;
}
```

### 6.2 删除表情

```dart
Future<void> deleteEmoji(String id) async {
  final emoji = _allEmojis[id];
  if (emoji == null) return;

  // 1. 删除本地文件
  if (emoji.localPath != null) {
    final file = File(emoji.localPath!);
    if (await file.exists()) {
      await file.delete();
    }
  }

  // 2. 删除备份文件
  if (emoji.backupPath != null) {
    await EmojiFileBackupService.deleteBackup(emoji.backupPath!);
  }

  // 3. 从所有角色的订阅中移除
  final roles = _contactProvider.allRoles;
  for (final role in roles) {
    if (role.subscribedEmojiIds.contains(id)) {
      await _contactProvider.unsubscribeEmoji(role.id, id);
    }
  }

  // 4. 删除数据库记录
  await (db.delete(db.emojis)..where((e) => e.id.equals(id))).go();

  // 5. 更新内存缓存
  _allEmojis.remove(id);
  notifyListeners();
}
```

### 6.3 表情显示组件

```dart
// lib/widgets/emoji_image_widget.dart
class EmojiImageWidget extends StatelessWidget {
  final EmojiModel emoji;
  final double size;

  const EmojiImageWidget({
    required this.emoji,
    this.size = 100,
  });

  @override
  Widget build(BuildContext context) {
    return FutureBuilder<String?>(
      future: _resolveImagePath(),
      builder: (context, snapshot) {
        if (snapshot.connectionState == ConnectionState.waiting) {
          return SizedBox(
            width: size,
            height: size,
            child: const CircularProgressIndicator(),
          );
        }

        final imagePath = snapshot.data;
        if (imagePath == null) {
          return _buildErrorWidget();
        }

        return Image.file(
          File(imagePath),
          width: size,
          height: size,
          fit: BoxFit.contain,
          errorBuilder: (_, __, ___) => _buildErrorWidget(),
        );
      },
    );
  }

  Future<String?> _resolveImagePath() async {
    // 1. 尝试主路径
    if (emoji.localPath != null) {
      final file = File(emoji.localPath!);
      if (await file.exists()) {
        return emoji.localPath;
      }
    }

    // 2. 尝试备份路径
    if (emoji.backupPath != null) {
      final file = File(emoji.backupPath!);
      if (await file.exists()) {
        // 从备份恢复到主路径
        if (emoji.localPath != null) {
          await EmojiFileBackupService.restoreFromBackup(
            emoji.backupPath!,
            emoji.localPath!,
          );
          return emoji.localPath;
        }
        return emoji.backupPath;
      }
    }

    return null;
  }

  Widget _buildErrorWidget() {
    return Container(
      width: size,
      height: size,
      color: Colors.grey[300],
      child: const Icon(Icons.broken_image),
    );
  }
}
```

## 7. 设计决策

### 7.1 为什么使用统一池？

**早期设计（已废弃）**:
```
角色 A 的表情存储
角色 B 的表情存储
角色 C 的表情存储
```

**问题**:
- 相同表情重复存储
- 难以实现跨角色分享
- "偷图"功能实现复杂

**当前设计**:
```
统一表情池 → 角色通过订阅使用
```

**优点**:
- 存储高效
- 订阅灵活
- 偷图自然

### 7.2 为什么使用文件而非 Blob？

| 对比项 | Blob 存储 | 文件存储 |
|--------|-----------|----------|
| 数据库大小 | 膨胀严重 | 保持轻量 |
| 内存占用 | 加载时高 | 按需加载 |
| 查询性能 | 随数据量下降 | 稳定 |
| 备份/恢复 | 需要解码 | 直接复制 |
| GIF 支持 | 需要处理 | 原生支持 |

**选择**: 文件存储 + 路径引用

### 7.3 为什么需要 backupPath？

**场景**: 用户清理应用缓存，导致 `localPath` 文件丢失

**解决方案**: 双路径策略
1. `localPath` - 主存储路径（可能被清理）
2. `backupPath` - 备份路径（应用私有目录，不会被清理）

**恢复流程**:
```
显示表情
    │
    ├── localPath 存在? → 显示
    │
    └── localPath 不存在?
            │
            ├── backupPath 存在? → 恢复到 localPath → 显示
            │
            └── 都不存在? → 显示错误图标
```

### 7.4 为什么回收 ID？

**场景**: 用户频繁添加/删除表情

**问题**: 如果只递增 ID，会产生大量空洞

```
添加: 00001, 00002, 00003
删除 00002
添加: 00004 (不回收) → 00002 永久浪费
```

**解决**: 回收已删除的 ID

```
添加: 00001, 00002, 00003
删除 00002
添加: 00002 (回收) → 无浪费
```

### 7.5 meaning vs rawContent

| 字段 | 用途 | 示例 |
|------|------|------|
| meaning | 简短关键词，用于 AI 匹配 | "开心" |
| rawContent | 详细描述，区分相似表情 | "笑得很开心，眼睛弯成月牙形" |

**AI 匹配优先级**:
1. meaning 精确匹配
2. meaning 包含匹配
3. rawContent 包含匹配
