# UI 规范：虚拟系统桌面 (Virtual OS Home)

本规范定义了“汪汪机”核心界面——虚拟桌面的视觉与交互细节。目标是实现 1:1 复刻 iOS 体验。

## 1. 布局逻辑 (Layout Logic)

### 1.1 虚拟状态栏 (Virtual Status Bar)
*   **位置:** 屏幕顶部，高度适配宿主设备刘海/挖孔，但需预留边距。
*   **内容:**
    *   左侧: 虚拟时间 (由 C++ Core 计算)。
    *   右侧: 虚拟信号强度 (5G图标), 虚拟电量 (电池图标 + 百分比)。
*   **状态:** 需与 C++ Core 中的 `SystemState` 绑定。

### 1.2 应用网格 (App Grid)
*   **布局:** 4 列 x N 行。
*   **间距:** 严格遵循 iOS 栅格系统 (参考: 水平间距 27pt, 垂直间距 30pt, 具体视屏幕宽度动态调整)。
*   **分页:** 支持左右滑动分页，带有底部 PageControl (小圆点)。

### 1.3 底部 Dock 栏 (Glass Dock)
*   **视觉:** 磨砂玻璃效果 (Gaussian Blur)。
    *   Android: 使用 `RenderEffect` (API 31+) 或 Shader 实现高性能模糊。
    *   iOS: 使用 `UIVisualEffectView`。
*   **内容:** 固定放置 4 个核心应用 (电话, 信息, Safari/浏览器, 音乐)。
*   **层级:** 位于所有应用图标之上。

### 1.4 小组件 (Widgets)
*   **尺寸:** 2x2 (小), 4x2 (中), 4x4 (大)。
*   **混合排布:** 必须支持 Widget 与 App Icon 的自动重排。

## 2. 视觉细节 (Visual Details)

### 2.1 动态壁纸与沉浸感
*   **层级:** 壁纸层 (Live2D/Video) -> 图标层 -> 状态栏/Dock 层。
*   **交互:** 这里的 Live2D 角色 (汪汪) 会对用户的触摸 (如点击图标) 做出视线跟随或躲避反应。

## 3. 手势处理 (Gesture Handling)

### 3.1 冲突解决 (Conflict Resolution)
*   **问题:** 虚拟 OS 的“上滑回到桌面”手势可能与宿主机的系统手势冲突。
*   **策略:**
    *   **边缘保护:** 在屏幕底部边缘保留宿主系统手势区。
    *   **虚拟 Home 条:** 在 Dock 栏下方绘制一个虚拟 Home Indicator，该区域响应虚拟系统手势。
